Testtool Description

1. Cachebench : CacheBench is a benchmark designed to evaluate the performance of the memory hierarchy.
Test cases executed by this tool: bandwidth

2. Compile: Compiling a kernel in multiple threads can evaluate the multi core/multi threading capability of the system.This can be used to measure how well the platform is scaling with respect to the number of cores (equivalent to the number threads of compilation)
Test cases executed by this tool: kernel-dev_#

3. Coremark: CoreMark is a benchmark that aims to measure the performance of central processing units (CPU) used in embedded systems

4. Dhrystone: Dhrystone is a synthetic computing benchmark program developed in 1984 by Reinhold P. Weicker intended to be representative of system (integer) programming

5. Fio: This is the most powerful IO benchmarking tool available today which is very flexible to create various io workloads and to execute them.
Test cases executed by this tool: bandwidth and iops

6. Hadoop: Hibench measures the common hadoop performance parameters like HadoopTerasort, HadoopWordcount, HadoopBayes, HadoopKmeans, HadoopSort, HadoopDfsioe etc.

7. Iozone: This is a filesystem benchmark tool. The benchmark tests file I/O performance for the following operations. Read, write, re-read, re-write, read backwards, read strided, fread, fwrite, random read/write etc
Test cases executed by this tool:Iozone-Cached and Iozone-DirectIO

8. Linpack: LINPACK is a collection of Fortran subroutines that analyze and solve linear equations and linear least-squares problems.

9. Lmbench : lmbench is a series of micro benchmarks intended to measure basic operating system and hardware system metrics.
Test cases executed by this tool: local_speed, lb_bw_stream_v#, lb_bw_cross_#_core, lb_bw_local_#_core, lb_lat_cross_#_core, lb_lat_local_#_core

10. Nbench: NBench is a synthetic computing benchmark program developed in the mid-1990s, intended to measure a computer's CPU, FPU, and Memory System speed.

11. Openblas: This tool is a set of low-level routines for performing common linear algebra operations such as vector addition, scalar multiplication, dot products, linear combinations, and matrix multiplication
Test cases executed by this tool: openblas_value_#_cores and openblas_efficiency_#_cores

12. Openssl: OpenSSL is a cryptography toolkit implementing the Secure Sockets Layer ( SSL v2/v3) and Transport Layer Security ( TLS v1) network protocols and related cryptography standards required by them.
All the test cases described here has been executed by openssl tool (except unzip test case).

13. Scimark: Measure sthe CPU performance.

14. JDK: Uses SciMark2 to evaluates the Java performance for scientific and numerical computing.It consists of five computational kernels: FFT, Gauss-Seidel relaxation, Sparse matrix-multiply, Monte Carlo integration, and dense LU factorization.Reports a composite score in approximate Mflops (Millions of floating point operations per second).

15. Stress-ng: Measure sthe CPU performance by running some CPU intensive routines.

16. Sysbench: sysbench oltp benchmark evaluate the database performance.
Test cases executed by this tool: max, avg, min and percentile

17. Tinymembench: This is a simple memory benchmark program, which tries to measure the peak bandwidth of sequential memory accesses and the latency of random memory accesses.
Test cases executed by this tool: Tiny latency and Tiny bandwidth

18. Unixbench: A system benchmark which test the performance of CPU and memory.
Test cases executed by this tool:multicore_unixbench

19. Unzip: This tool is to measure the time required to extract the tar.gz file.

20. Nginx: Nginx tool is mainly used to measure the web server performance.
Test cases executed by this tool: nginx_32_core_local_wrps, nginx_32_core_cross_wrps and nginx_64_core_wrps

21. Redis: Redis tool evaluate the performance of database. Redis works with an in-memory dataset.
Test cases executed by this tool: redis_Instance_#

22. Netperf : Netperf is a benchmark that can be used to measure various aspects of networking performance.
Test cases executed by this tool: TCP_RR, TCP_stream_r, TCP_CRR, TCP_stream and UDP_RR

23. Iperf : Iperf is a commonly-used network testing tool that can create Transmission Control Protocol (TCP) and User Datagram Protocol (UDP).
Test cases executed by this tool: TCP_s#_RX and TCP_s#_TX

24. Qperf: qperf measures bandwidth and latency between two nodes. It can work over TCP/IP as well as the RDMA transports.
Test cases executed by this tool: TCP_bw_#B and TCP_lat_#B
